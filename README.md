# 日本株バックテスト＆ペーパートレードアプリ

Yahoo Finance から日本株データを取得し、売買戦略の検証・仮想売買・勝因敗因分析・
過剰最適化チェック・自動売買・定期レビューまでを行う Web アプリです。5つの画面で構成されています。

| 画面 | URL | 内容 |
|------|-----|------|
| バックテスト | `/` | 戦略を1回検証＋パラメータ総当たりで利益条件を探索 |
| 調査 | `/screen` | 日米株から数日〜2週間で上がりそうな銘柄をスコア順に一覧（期間別の上昇確率・期待幅つき） |
| ペーパートレード | `/paper` | 仮想資金で売買し、取引ごとに勝因・敗因を記録・自動分析 |
| 戦略検証 | `/validate` | ウォークフォワード検証で「将来も通用する戦略」を選別 |
| 自動売買 | `/live` | 検証◎の戦略だけを毎営業日16:30に自動で仮想売買（ボット）|
| 振り返り | `/report` | ボット成績を日経平均と比較し勝因敗因・改善提言を毎週レポート |

- **バックエンド**: FastAPI（自動売買は APScheduler でスケジュール実行）
- **フロントエンド**: htmx + Chart.js（ページ遷移なしで結果を差し替え）
- **データ**: yfinance（Yahoo Finance）＋ SQLite に永続蓄積
- **パッケージ管理**: uv

## 自動売買（`/live`）の使い方

1. **① 銘柄選定を更新** … 全銘柄×全戦略をウォークフォワード検証し、「信頼できる◎」と
   出た銘柄・戦略・パラメータだけをボットの取引対象に登録（1〜2分）。
2. **② 今日の分を手動実行** … 最新データを取得し、当日シグナルで建玉/手仕舞い。
3. アプリ起動中は、毎営業日（月〜金）**16:30 JST に自動実行**される。

口座（初期100万円・`live`）は実際の現金を取引ごとに増減し、建玉・含み損益・資産推移・
日次売買ログを画面で確認できる。**実際の発注はしないペーパートレード**。約定はその日の終値。

## 調査スクリーナー（`/screen`）

日米計約210銘柄（`app/stocks.py` の `UNIVERSE`）を価格・出来高という一次データだけから評価し、
**数日〜2週間（3営業日 / 1週間 / 2週間）で上がりやすい銘柄**を総合スコア順に一覧する。誰かの推奨の
引用ではなく、システム自身の判断を**過去データで較正した的中率・期待リターン幅つき**で提示する。

使い方（はじめは ② → ① の順）:

1. **② スコアを較正** … 過去5年のデータで「スコア帯ごとに実際どれだけ上がったか」を期間別に集計し、
   `screen_calibration` テーブルへ保存（重いので数分かかる）。これにより総合スコアが
   **上昇確率(%)・期待リターン分布(25%〜中央〜75%)** に変換される。
2. **① 最新データで再評価** … 全銘柄の最新株価を取得して総合スコアを計算し、`screen_snapshots` に保存。

各行に表示する内容: 総合スコア・上昇確率・期待リターン幅・想定エントリー/損切り(ATRベース)/利確目標・
想定保有日数・決算警告（対象期間内に決算予定があれば ⚠）・選定根拠。期間タブ（3日/1週間/2週間）で切替。

スコアの構成要素（`app/screener.py`）: トレンド（75日線乖離）・モメンタム（20日リターン＋MACD）・
押し目の質（RSIが過熱でない）・対指数の相対強さ・出来高の盛り上がり。各要素を同一日の全銘柄間で
ランク正規化して加重合成し、下落相場では順張りスコアを減衰、流動性の薄い銘柄は除外する。

画面下部の**的中率テーブル**は「スコア上位帯ほど実際に上昇確率が高いか」を可視化したもので、
スコアが機能しているかの裏取りになる。短期の値動き予測は本質的に難しく、確率が50%台に収れんすることも
多い。表示は過去分布にもとづく推定であり将来を保証しない（投資は自己責任）。

`/screen` は3つのタブで構成される。

### 候補一覧
上記のスコア順一覧。「①再評価」で最新評価を保存（`screen_snapshots`）、「②較正」で確率・期待幅を更新。

### 精度（記録）
予測がどれだけ当たったかを**結果論を排して**検証する。

- **再計算（ウォークフォワード検証）** … 過去を時系列フォールドに分け、各フォールドを
  「その開始より前のデータだけ」で較正・予測して実績と照合（`app/calibration.py: walk_forward_accuracy`）。
  未来情報の混入が無いので、即座に「予測確率 vs 実績の上昇割合」が分かる。予測と実績が一致（対角線）ならスコアは信頼でき、
  乖離があれば過信／過小評価＝重み調整など改善の手がかりになる。
- **記録（実績を記入）** … 期限が到来した過去の予測に実際の結果を埋める（`app/outcomes.py: fill_outcomes`）。
  建玉は予測日の**翌営業日始値**、損切り・利確・期限は予測時に固定した値で前向きに判定する。
  実運用のトラックレコードとして日が経つほど蓄積される。

### シミュレーション
「もしスコア上位を機械的に売買していたら」をフォワード（後出しなし）で再現（`app/screen_sim.py`）。
各時点の終値までで決まる因果的スコアで上位 N 銘柄を選び、**翌営業日始値**で均等建玉、
損切り(ATR×2)／利確(リスクの1.5倍)／期限満了のいずれか先着で手仕舞う。期間・保有日数・銘柄数・リバランス頻度を選べる。
エクイティ曲線（対 日米指数均等ブレンド）・勝率・最大DD・シャープ・トレード明細を表示する。
**同じ条件なら結果は常に一致**（決定論）＝あと出しの余地が無いことの担保。

これらは毎営業日16:45（予測保存＋実績記入）・毎週金曜17:30（精度再計算）にも自動実行される（`app/scheduler.py`）。

## 振り返りレポート（`/report`）

ボットの運用成績を**日経平均と比較**し、確定取引の勝因敗因・保有中ポジション講評・
**改善提言**（ベンチマーク割れ・不調戦略・下落相場での敗北・過大ドローダウン等を検知）を
レポート化する。毎週金曜17:00 JSTに自動生成・保存され、履歴として時系列で見返せる。

- 提言は**観察ベースで投資助言ではない**。再選定などの変更は人間がボタンで判断（自動実行しない）。
- 対ベンチマーク比較はボット稼働期間のみが対象。運用初期は確定取引が少なくレポートは薄い。

## 起動方法

```bash
./run.sh
```

または直接:

```bash
uv run uvicorn app.main:app --reload --port 8000
```

起動後、ブラウザで <http://127.0.0.1:8000> を開きます。

## 常時起動での自動運用

予測の保存・実績記入・較正は、PC を起動しておけば自動で回ります（Web サーバの起動は不要）。

- **macOS**: `bash scripts/install_launchd.sh` で launchd に登録（平日 07:30/16:45 にスクリーニング、
  土 18:00 に精度再計算）。解除は `bash scripts/uninstall_launchd.sh`。
- **Windows（研究室の常時起動PC向け・推奨）**: タスクスケジューラ版を `scripts/windows/` に用意。
  セットアップ手順は **[scripts/windows/README.md](scripts/windows/README.md)** を参照
  （git+uv 導入 → `git clone` → `uv sync` → `register_tasks.ps1` → サーバ常駐 → 電源/タイムゾーン設定）。

リモートで使う場合は **Tailscale**（ポート開放なしで安全に `http://<PC>:8000` へ）を主軸に、
画面操作が必要なときだけ RDP / Chrome リモートデスクトップを併用するのが安全です。
本アプリは**認証が無い**ため、公開インターネットへ直接ポート開放しないでください。

> 注意: Mac と Windows の両方で自動実行すると実績DB（`data/stock.db`）が分裂します。
> 常時起動機を1台に決め、もう一方の自動実行は停止してください。

## 使い方

1. **銘柄・戦略・期間**を選ぶ。
2. **「バックテスト実行」** … 既定パラメータで1回検証。資産推移グラフ・勝率・
   最大ドローダウン・取引履歴を表示。
3. **「利益が出る条件を探索」** … パラメータの全組み合わせを総当たりし、
   累積リターンの高い順にランキング表示。

## 実装している戦略

| 戦略 | 内容 |
|------|------|
| 移動平均クロス | 順張り。短期SMAが長期SMAを上回れば保有、下回れば現金 |
| RSI逆張り | 逆張り。RSIが下限以下で買い、上限以上で売り |
| ブレイクアウト | 順張り。過去N日高値更新で買い、N日安値割れで売り |
| MACD | 順張り。MACD線がシグナル線を上抜けで保有、下抜けで現金 |
| ストキャスティクス | 逆張り。%Dが下限以下で買い、上限以上で売り |
| ボリンジャーバンド | 逆張り。終値が-σバンド割れで買い、+σバンド超えで売り |

## 出口ルール（手仕舞い）

戦略の売りシグナルに加え、`app/exits.py` で以下の出口ルールを併用できる（`/paper` の
入力欄で数値を調整して実験、`/live` の設定は自動売買ボットに永続適用）。

| ルール | 内容 |
|--------|------|
| 固定損切り | 含み損が -X% で手仕舞い |
| 固定利確 | 含み益が +X% で手仕舞い |
| トレーリングストップ | 建玉後の高値から -X% 下落で手仕舞い |
| ATRベース損切り | 建玉時のATR×倍率を損切り幅にしボラに自動適応 |

優先順位は 利確 → トレーリング → ATR損切り → 固定損切り。ジャーナルの「出口別成績」で
どの手仕舞いが効いているかを確認できる。

## バックテストの前提

- シグナルは **翌営業日の始値** で約定（先読みバイアス回避）。
- 取引コスト（手数料＋スリッページ）を片道 0.1% で計上。
- ロングのみ（空売りなし）。
- 配当を含む調整後株価（`auto_adjust=True`）を使用。

## 対象銘柄（10銘柄）

`app/stocks.py` で定義。トヨタ・ソニー・ソフトバンクG・キーエンス・
三菱UFJ・NTT・リクルート・信越化学・東京エレクトロン・任天堂。
証券コードを追加すれば銘柄を増やせます。

## ディレクトリ構成

```
app/
├── main.py          # FastAPI ルーティング
├── data.py          # yfinance データ取得（キャッシュ付き）
├── stocks.py        # 銘柄ユニバース（日本株 STOCKS／日米統合 UNIVERSE）
├── strategies.py    # 売買シグナル生成（戦略）
├── backtest.py      # バックテスト＋パラメータ探索エンジン
├── screener.py      # 調査スクリーナー（総合スコア・上昇候補の抽出）
├── calibration.py   # スコアの較正＋精度検証（ウォークフォワード/実運用）
├── outcomes.py      # 予測の実績照合（記録）と共通の前向き出口判定
├── screen_sim.py    # フォワード・シミュレーション（結果論なしの売買検証）
└── templates/       # htmx 用 HTML
```
